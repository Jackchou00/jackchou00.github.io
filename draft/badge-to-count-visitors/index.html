需求背景#

想为博客添加一个visitor badge来显示访客统计，要求显示在footer下方居中位置，而不是混在footer文字中。

实现方案#

技术选择#

使用 visitor-badge.laobi.icu 服务来生成访客统计徽章，这是一个免费的第三方服务，只需要提供页面ID即可生成美观的SVG徽章。

修改过程#

第一步：配置分离#

将visitor badge从footer文字中分离出来，在hugo.yaml中单独配置：

文件位置： hugo.yaml:82-85

footer:
  hideCopyright: false
  text: "my friends' websites: [zhxwu](https://zhxwu.com/)"
visitorsImage: "https://visitor-badge.laobi.icu/badge?page_id=page.id"

修改说明：

  • 从footer.text中移除了![visitors](URL)代码
  • 新增visitorsImage参数专门存储badge URL
  • 保持footer文字的简洁性

第二步：主题扩展#

修改PaperMod主题的extend_footer.html文件来显示badge：

文件位置： themes/PaperMod/layouts/partials/extend_footer.html:3-7

{{- if .Site.Params.visitorsImage -}}
<div style="text-align: center; margin-top: 10px; margin-bottom: 20px; width: 100%; display: flex; justify-content: center;">
    <img src="{{ .Site.Params.visitorsImage }}" alt="visitors" style="max-width: 120px; height: auto;" />
</div>
{{- end -}}

技术细节#

Hugo模板语法#

  • {{- if .Site.Params.visitorsImage -}}：条件判断，只有配置了visitorsImage才显示
  • {{ .Site.Params.visitorsImage }}：读取hugo.yaml中的配置参数
  • {{- -}}：去除多余的空白字符

CSS样式设计#

  • text-align: center：文本居中对齐
  • width: 100%：容器占满父元素宽度
  • display: flex; justify-content: center：Flexbox布局确保元素水平居中
  • margin-top: 10px; margin-bottom: 20px：与footer保持适当间距，底部留白
  • max-width: 120px：限制badge最大宽度，避免过大显示
  • height: auto：保持图片比例

布局考虑#

  1. 位置选择：使用extend_footer.html而不是修改footer.html，避免主题更新时丢失修改
  2. 响应式：使用百分比和Flexbox确保在不同屏幕尺寸下都能正确居中
  3. 兼容性：Hugo模板条件判断确保没有配置时不会显示空白区域

优势分析#

与直接在footer.text中显示相比#

  1. 布局控制：可以精确控制位置、大小和间距
  2. 配置灵活：更换URL只需修改一个参数
  3. 主题兼容：使用主题提供的扩展点，升级主题时不会丢失
  4. 样式独立：不受footer文字样式影响

维护便利性#

  1. 参数化配置：更换统计服务或page_id只需修改hugo.yaml
  2. 开关控制：删除visitorsImage参数即可隐藏badge
  3. 主题安全：不直接修改主题核心文件

使用说明#

更换统计链接#

修改hugo.yaml中的visitorsImage参数：

visitorsImage: "https://visitor-badge.laobi.icu/badge?page_id=jackchou.top"

调整样式#

修改extend_footer.html中的CSS样式：

  • 调整大小：修改max-width值
  • 调整间距：修改margin-top和margin-bottom值
  • 调整位置：修改Flexbox属性

移除功能#

删除或注释掉hugo.yaml中的visitorsImage参数即可。

注意事项#

  1. 主题更新：如果更新PaperMod主题，需要重新添加extend_footer.html的修改
  2. 第三方依赖：依赖外部统计服务，如果服务不可用会显示破图
  3. 隐私考虑：访客统计会收集用户访问数据，需要符合隐私政策要求

扩展可能#

这个方案可以轻松扩展到其他类型的徽章，比如：

  • GitHub stars badge
  • License badge
  • Build status badge
  • 等等，只需要在hugo.yaml中添加相应参数，在extend_footer.html中添加对应的显示逻辑即可。
